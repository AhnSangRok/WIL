## 회고 - 미니 프로젝트 - Spring (07.25~28)
저번주에 이어 프로젝트를 진행했고 부분부분 에러가 나오기는 했지만 간단하게 고칠 수 있었다. 어느정도 기능이 완성되서 이제 프론트와 합치려고 
하는데 CORS가 발목을 잡았다. CORS의 정책 위반으로 인해 에러가 발생되어 데이터를 받아와도 사용하지 못하는 것이다. 덕분에 하루를 그렇게 넘기고
다음날 security config 파일 안에 CORS를 선언해서 모든 요청을 허락하도록 한 후에 겨우 데이터를 내려 받을 수 있었다. 하지만 시간 부족으로 
로그인 기능을 넣지못해 미완성인 상태로 제출 할 수 밖에 없었다. 
#### 배웠고 느꼈던 점
프론트와 백엔드 각각 작업하고 합치는 과정이 쉽지 않을 것이라는 생각은 했지만 현실은 더 했던것 같다. 처음부터 프론트와 백엔드의 역할에 대해서 
좀 더 고민하게 되는 프로젝트였다. 일단 Api 수정시에 데이터 이름이나 url 같은 경우 빠르게 전달해서 이름이 맞지않아 오류를 일으키는 일이 없도록
해야한다는 것, 물론 당연한 일이겠지만 하다보니 시아가 좁아져 미쳐 생각이 나지 않았다. 다음부터는 주의를 해야겠다. 그리고 프로젝트의 완성을
하지 못해서 아쉬웠다. 몰랐던 부분과 서툰 부분에 대해서 몸으로 직접 느끼니 오히려 의욕이 나는 것 같다. 그 부분을 채우면 좀 더 성장할 수 있다고
조심스레 생각해본다. 마지막으로 프론트 부분에 대해서도 어떤 일을 하는지 대략적으로 파악하는 것이 좋을 것 같다. 어느 부분에서 백엔드와 프론트의
의견을 나눠야 하는지 서로 처리할 수 있는 부분에 대해서는 회의를 통해 적절하게 분배를 할 수 있도록 하면 서로 좋게 프로젝트를 마칠 수 있을 것
같다.

## 회고 - 클론코딩 - Spring (07.29~31)
새로운 프로젝트 클론코딩이 시작됐다. 우리팀은 마켓 컬리를 선택했고 주요 기능으로 장바구니와 물건들의 CRUD를 구현하는 것이다. 아직은 기능이
적지만 저번주의 교훈으로 일단 작은 기능부터 완벽하게 구현하고 점차 살을 붙어 가는 것으로 결정했다. 이번에는 프론트 분이 2명이라 다행이였지만
백엔드가 4명이라 상대적으로 많아서 프론트 분들이 무리가 가지 않는 선에서 백엔드의 기능을 좀 더 넣기로 했다. 와이어 프래임과 api, ERD를 
작성하고 역할 분담을 했다. 나는 장바구니를 맡았는데 이번 만큼은 완벽하게 구현할 것이라고 마음을 먹었다.
#### 배웠고 느꼈던 점
처음이 가장 중요하다는 것을 느꼈다. 시작을 하기전에 준비를 꼼꼼하게 했더니 지금까지는 잘 진행되고 있다. 물론 내가 맡은 장바구니는 아직 오류
투성이지만 월요일까지는 완성이 될 것 같다. JPA를 이용해 장바구니와 장바구니에 들어갈 물건들에 대한 연관 관계를 설정하고 구현하는게 아직 서툴
지만 이제는 코드를 보기만 해서는 풀리지 않는다는 것을 알고 있다. JPA에 대해 다시 공부하고 다시 구현할 생각이다. 급한 마음에 붙들고만 있었지만
이제는 다시 처음부터 돌아가야 할때를 알고 실행하는 것이 좀 더 유의미한 결과를 낳는다는 것을 여러 차례 프로젝트를 진행하면서 알게됐다. 추가로 
팀원분이 JWT에 대해 원래 사용하던 코드를 두고 다시 강의를 보면서 하나하나 짚어서 자신의 것으로 만들려는 모습을 보고 나도 저렇게 해야겠다는 
생각을 행동으로 옮기는데 큰 영향을 끼쳤다. 
이제 이번주 프로젝트가 끝나면 실전이다. 끝까지 포기하지 않고 나아가야 겠다.
